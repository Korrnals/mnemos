# Runbook: Обновление зависимостей и напоминание о CVE

**🌐 Language / Язык:** [English](../../../en/admin/runbooks/dependency-updates.md) · Русский

## Причина существования этого runbook'а

`chromadb` на данный момент имеет неисправленную `CVE-2026-45829`, поскольку
апстрим-релиза с фиксом ещё нет. Mnemos держит это видимым как предупреждение
в `make verify`, чтобы не забыть обновиться при появлении фикса.

## Политика закрепления версий (M15.5.1)

Mnemos использует **прямые pin'ы** для уязвимых транзитивных зависимостей вместо
обновления родительских пакетов:

- **`aiohttp>=3.14.1,<4.0`** — прямой pin, закрывает CVE-2026-34993, 47265,
  50269, 54273-54280. Подтягивается транзитивно через `chromadb → kubernetes`.
  Прямое закрепление безопасной минорной версии имеет меньший радиус взрыва,
  чем обновление `chromadb` (у которого нет вышедшего релиза с патчем aiohttp).

- **`starlette>=1.3.0,<2.0`** — прямой pin, закрывает CVE-2026-48817, 48818,
  54282, 54283. Подтягивается транзитивно через `fastapi`. Та же логика.

- **`pip` 26.1.2** — обновлять через `pip install --upgrade pip` после
  пересоздания venv. Закрывает PYSEC-2026-196. `pip` — инструмент, а не
  зависимость проекта, поэтому не включён в `pyproject.toml`.

- **`chromadb>=0.5`** — оставлен на существующем полу. Обновление chromadb
  **не** закрывает CVE aiohttp/starlette, поскольку эти пакеты не входят в
  прямые зависимости chromadb.

При добавлении нового pin'а: включить однострочный комментарий в `pyproject.toml`
с CVE id и версией фикса, как в существующих записях. Pin'ы должны использовать
диапазон с верхней границей (`<4.0`, `<2.0`) во избежание случайного major-дрейфа.

## Ежедневная/еженедельная быстрая проверка

```bash
cd /var/home/abyss/LABs/AI/mnemos
source .venv/bin/activate
make update-chromadb
```

Ожидаемые исходы:
- Если новой версии `chromadb` нет: ничего не меняется, напоминание остаётся активным.
- Если вышла исправленная версия: пакет обновляется и `pip-audit` должен стать
  чистым без ignore.

## Полное обновление зависимостей

```bash
cd /var/home/abyss/LABs/AI/mnemos
source .venv/bin/activate
make update-deps
```

Затем запустить полные проверки проекта:

```bash
make verify
```

## Удаление временного игнора CVE после фикса

Когда `pip-audit` подтверждает наличие и установку фикса:
1. Открыть [Makefile](../../../../Makefile)
2. В target'е `security` удалить `--ignore-vuln CVE-2026-45829`
3. В target'е `security-reminder` удалить строки предупреждения
4. Запустить:

```bash
make verify
```

## Операционная политика

- Держать ignore только пока апстрим не выпустил фикс.
- Держать напоминание активным, пока существует ignore.
- Удалить и ignore, и напоминание в одном коммите после обновления.
