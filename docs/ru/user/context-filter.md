# Руководство по контекстному фильтру

**🌐 Language / Язык:** [English](../../en/user/context-filter.md) · Русский

Контекстный фильтр — пятиступенчатый конвейер, который очищает сырой
контент от шума **до** того, как он попадёт к модели. Фильтр запускается
автоматически при каждом `mnemos_add` (когда `auto_filter: true`) и может
быть повторно применён к существующим записям через MCP-инструмент
`mnemos_filter` или CLI-команду `mnemos filter`.

---

## Что он делает

Сырой ввод агента — вывод терминала, логи сборки, скачанные веб-страницы —
часто в 5–10 раз больше того сигнала, который нужен модели. Фильтр стоит
между приёмом и хранением и формирует поле `clean_content`, которое поиск
и recall возвращают вместо сырого блоба.

| Стадия | Название | Что удаляется | Применяется к |
|--------|----------|---------------|---------------|
| 1 | **dedup** | Точные и почти-дубликаты строк (нормализованное сравнение) | все профили |
| 2 | **noise** | ANSI-коды, прогресс-бары, временные метки, линии-разделители | log, terminal, default |
| 3 | **extract** | Сигнальные строки (ошибки, предупреждения, коды выхода) + контекстное окно; сэмплирование для длинных логов | log, terminal |
| 4 | **compress** | Повторяющиеся блоки (стек-фреймы, повторяющиеся паттерны) сворачиваются в `... (N similar lines) ...` | все профили |
| 5 | **tokens** | Оценка токенов + опциональная truncation по бюджету | все профили |

Оригинальный контент никогда не теряется: `raw_content` сохраняется рядом
с `clean_content`, а `filter_stats` фиксирует метрики по каждой стадии.

---

## Профили

Фильтр выбирает профиль автоматически по эвристикам содержимого. Можно
задать явно через `--profile` (CLI) или аргумент `profile`
(MCP-инструмент).

| Профиль | Когда выбирается | Что оптимизирует |
|---------|------------------|------------------|
| `log` | ISO-временные метки или маркеры `[DEBUG\|INFO\|WARN\|ERROR]` в первых 20 строках | Логи — убрать временные метки, извлечь ошибки и предупреждения |
| `terminal` | Присутствуют ANSI-коды или глифы спиннера (`▶◀◐◑`) | Вывод интерактивного терминала — убрать ANSI, прогресс-бары |
| `code` | Паттерны `def` / `class` / `import` / `function` / `const` (≥3 совпадений) | Исходный код — дедупликация и сжатие, без удаления временных меток |
| `web` | Присутствуют теги `<html`, `<!doctype` или `<div` | HTML-контент — дедупликация и сжатие |
| `docs` | Markdown с множеством заголовков `#` или правил `---` | Документация — дедупликация и сжатие |
| `default` | Ни один из вышеперечисленных не сработал | Общий — дедупликация, шум, сжатие, без профильного извлечения |

### Порядок определения профиля

1. Явный аргумент `hint` / `--profile` (если валиден) → используется как есть.
2. `log` — временные метки или маркеры уровня лога.
3. `terminal` — ANSI-коды или глифы спиннера.
4. `code` — паттерны ключевых слов кода.
5. `web` — HTML-теги.
6. `docs` — структура markdown.
7. `default` — фолбэк.

## Автофильтр при приёме

Когда `auto_filter: true` (по умолчанию для новых установок), каждый
вызов `mnemos_add` запускает конвейер фильтрации перед сохранением записи.

```yaml
# config.yaml
mnemos:
  auto_filter: true
```

Что происходит при приёме:

1. Запись сохраняется с `raw_content` = исходный ввод.
2. Фильтр запускается на `raw_content`, формируя `clean_content`.
3. Сохраняются `filter_profile`, `filter_stats` и `filter_version`.
4. Если фильтр падает, запись **всё равно сохраняется** только с
   `raw_content` — ошибки фильтрации не фатальны.

`mnemos_search` и `mnemos_recall_context` возвращают `clean_content`, если
он есть, и падают на `raw_content` (или `content`), если запись сохранена
до включения фильтрации.

## Ручной фильтр

### CLI — одна запись

```bash
mnemos filter <memory-id>
```

Перезапускает фильтр на существующей записи. Автоопределяет профиль, если
не задан `--profile`. Выводит определённый профиль, метрики по стадиям и
результирующий `clean_content`.

```bash
mnemos filter abc123 --profile terminal
mnemos filter abc123 --budget 2000
```

| Флаг | Описание |
|------|----------|
| `--profile`, `-p` | `log \| terminal \| code \| docs \| web \| default` (автоопределение, если опущен) |
| `--budget`, `-b` | Бюджет токенов для стадии truncation |
| `--all` | Перефильтровать все записи (выводит агрегированные метрики) |
| `--config` | Путь к файлу конфигурации (стандартный параметр) |

### CLI — все записи

```bash
mnemos filter --all
```

Итерирует все записи батчами и пере применяет фильтр. Полезно после
включения `auto_filter` на хранилище с уже нефильтрованными записями или
после обновления конвейера. Выводит счётчики `filtered`, `total`,
`failed` и `skipped`. Ошибки отдельных записей не фатальны.

### MCP-инструмент — `mnemos_filter`

Агенты могут вызывать `mnemos_filter` явно, чтобы перефильтровать запись —
например, когда автоопределённый профиль ошибся или нужно применить новый
профиль.

| Параметр | Тип | Обязателен | По умолчанию | Описание |
|----------|-----|------------|--------------|----------|
| `memory_id` | string | **да** | — | ID записи для фильтрации |
| `profile` | string | нет | автоопределение | `log \| terminal \| code \| docs \| web \| default` |
| `budget` | integer | нет | нет | Бюджет токенов для truncation |

Возвращает:

```json
{
  "memory_id": "abc123",
  "profile": "terminal",
  "clean_content": "...отфильтрованный текст...",
  "stats": {
    "profile": "terminal",
    "dedup": { "lines_in": 120, "lines_out": 98, "exact_dups": 18, "near_dups": 4 },
    "noise": { "original_chars": 8400, "removed_ansi": 32, "removed_progress": 12, "removed_timestamps": 20, "removed_separators": 3 },
    "extract": { "signal_lines": 15, "sampled": 0 },
    "compress": { "original_lines": 98, "compressed_lines": 90, "compressed_blocks": 2 },
    "tokens": { "estimated_tokens": 410, "budget": null, "truncated": false }
  }
}
```

## Когда использовать явный фильтр

- **Неверный авто-профиль** — лог определён как `default`; перезапустите с
  `--profile log`, чтобы убрать временные метки и извлечь сигналы.
- **Старые нефильтрованные записи** — записи, добавленные до включения
  `auto_filter`, не имеют `clean_content`. Запустите `mnemos filter --all`
  для бэкфилла.
- **Новый профиль** — после обновления конвейера с новым профилем,
  перефильтруйте, чтобы воспользоваться улучшенными эвристиками.
- **Изменение бюджета** — перезапустите с `--budget`, чтобы задать более
  жёсткий лимит токенов для конкретного контекста recall.

## Метрики фильтра в `mnemos stats`

`mnemos stats` включает секцию фильтра со здоровьем хранилища:

```text
auto_filter: True
filtered_count: 142
unfiltered_count: 8
avg_reduction_pct: 63.4
by_profile: {'log': 48, 'terminal': 52, 'code': 22, 'default': 20}
```

| Метрика | Значение |
|---------|----------|
| `auto_filter` | Включён ли автофильтр в конфигурации |
| `filtered_count` | Записи с заполненным `clean_content` |
| `unfiltered_count` | Записи без `clean_content` (до фильтра или с ошибкой) |
| `avg_reduction_pct` | Среднее сокращение размера по отфильтрованным записям |
| `by_profile` | Количество записей по определённым профилям |

Высокий `unfiltered_count` относительно `filtered_count` — повод запустить
`mnemos filter --all` для бэкфилла.

## Примеры

### Вывод терминала — до и после

**Сырой вывод (340 строк, 12 000 символов):**

```text
[38;5;10m✓[0m Building...
[38;5;10m✓[0m Building...
[██████████] 100%
2026-06-20T10:14:32Z INFO  Starting build
2026-06-20T10:14:33Z INFO  Compiling module A
2026-06-20T10:14:34Z ERROR FileNotFoundError: config.yaml
2026-06-20T10:14:34Z INFO  Compiling module B
...
```

**Отфильтровано (профиль `terminal`, 42 строки, 1 800 символов):**

```text
✓ Building...
ERROR FileNotFoundError: config.yaml
INFO  Compiling module B
... (8 similar lines) ...
```

ANSI-коды, прогресс-бары и временные метки убраны; дубликаты строк сборки
свёрнуты; строка с ошибкой сохранена с контекстом.

### Код — до и после

**Сырой вывод (повторяющиеся блоки импортов по файлам):**

```python
import os
import sys
import logging
import os
import sys
import logging
```

**Отфильтровано (профиль `code`):**

```python
import os
import sys
import logging
```

Точные дубликаты удалены; детектор почти-дубликатов нормализует строки
(сравнение только по алфавитно-цифровым символам), поэтому варианты,
отличающиеся только пробелами, тоже сворачиваются.

## Конфигурация

```yaml
mnemos:
  auto_filter: true   # запускать фильтр при каждом mnemos_add (по умолчанию: true)
```

Чтобы отключить автофильтр (хранить только сырой контент, фильтровать
вручную позже):

```yaml
mnemos:
  auto_filter: false
```

Выбор профиля автоматический — глобального «профиля по умолчанию» в
конфиге нет. Переопределение на каждый вызов через `--profile` (CLI) или
аргумент `profile` (MCP `mnemos_filter`).
