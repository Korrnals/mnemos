# Экспорт и импорт

**🌐 Language / Язык:** [English](../../en/user/export-import.md) · Русский

> Резервное копирование, миграция и восстановление памятей Mnemos через
> CLI или HTTP API. JSON-экспорт содержит метаданные (векторы
> регенерируются при импорте); SQLite-экспорт — полный снимок. Трассы
> никогда не экспортируются — это журналы аудита, а не данные памяти.

---

## Обзор

Подсистема экспорта/импорта позволяет:

- **Создавать резервные копии** хранилища памятей в портативный файл
  (JSON или SQLite).
- **Мигрировать** памяти между инстансами Mnemos.
- **Восстанавливать** предыдущее состояние после потери данных или
  неудачного импорта.
- **Фильтровать** экспортируемые памяти (по проекту, агенту, статусу,
  тегам, диапазону дат).
- **Шифровать** экспорт паролем (AES-256-GCM, деривация ключа через
  PBKDF2).
- **Сжимать** экспорт gzip для экономии места.
- **Запускать инкрементальные** бэкапы через `--since` для периодических
  снимков.

Доступны две поверхности: CLI-команды `mnemos export` / `mnemos import`
и HTTP-эндпоинты `POST /api/v1/export` / `POST /api/v1/import`. Обе
используют общую логику.

## Форматы экспорта

| Формат | Что содержит | Векторы | Трассы | Для чего |
|--------|--------------|---------|--------|----------|
| **JSON** (`--format json`) | Метаданные памятей + проекты | Регенерируются при импорте | Никогда | Частичный экспорт, миграция, инспекция |
| **SQLite** (`--format sqlite`) | Сырые `mnemos.db` + `vectors.db` в `.tar.gz` | Включены (полный снимок) | Никогда | Быстрый полный бэкап / восстановление |

### JSON — только метаданные

JSON-экспорт содержит метаданные каждой памяти (контент, теги, статус,
временные метки, проект, агент) и реестр проектов. **Векторы не
включаются** — они регенерируются при импорте путём повторного
эмбеддинга опубликованных памятей. Это сохраняет файл экспорта
компактным и переносимым.

### SQLite — полный снимок

SQLite-экспорт копирует сырые файлы `mnemos.db` и `vectors.db` в архив
`.tar.gz`. Это самый быстрый способ создать полный бэкап и самый быстрый
способ его восстановить (файлы БД заменяются напрямую). Фильтры
(`--project`, `--agent` и т.д.) **не применяются** к SQLite-экспорту —
это всегда полный снимок.

### Что никогда не экспортируется

- **Трассы** — таблица `traces` хранит журналы аудита пайплайна (шаги
  cluster, synthesize, publish, recall с задержками и LLM-флагами). Это
  операционные журналы, а не данные памяти, и они исключены из каждого
  формата экспорта по дизайну.

## Частичный экспорт — фильтры

Фильтры применяются **только к JSON-экспорту**. SQLite-экспорт всегда
полный.

| Фильтр | Флаг | Совпадение |
|--------|------|------------|
| Проект | `--project <slug>` | Памяти с `project:<slug>` |
| Агент | `--agent <slug>` | Памяти с `agent:<slug>` |
| Статус | `--status <value>` | `raw`, `processing`, `processed`, `published`, `archived` |
| Теги | `--tags a,b,c` | Памяти, содержащие все указанные теги (логика И) |
| С | `--since 2026-06-01` | Памяти, созданные/обновлённые после этой ISO-даты |
| По | `--until 2026-06-20` | Памяти, созданные/обновлённые до этой ISO-даты |

Пример — экспорт только опубликованных памятей проекта `mnemos`:

```bash
mnemos export \
  --format json \
  --project mnemos \
  --status published \
  --output mnemos-published.json
```

## Сжатие

```bash
mnemos export --format json --compress gzip --output backup.json.gz
```

| Режим | Флаг | Примечания |
|-------|------|------------|
| Без сжатия | `--compress none` (по умолчанию) | Нет сжатия |
| Gzip | `--compress gzip` | Стандартная библиотека, доступна везде |
| Zstd | `--compress zstd` | Сообщается как будущая возможность, если опциональная зависимость отсутствует |

## Шифрование

```bash
mnemos export --format json --encrypt --output backup.enc
# Passphrase: ******** (запрашивается, скрыт, с подтверждением)
```

Шифрование использует **AES-256-GCM** с ключом, выведенным из пароля
через **PBKDF2** (200 000 итераций, 16-байтная соль). Соль и nonce
добавляются в начало шифртекста, поэтому файл самодостаточен для
расшифровки.

| Источник пароля | Флаг | Когда использовать |
|-----------------|------|---------------------|
| Интерактивный ввод | (по умолчанию при `--encrypt`) | Ручные бэкапы |
| Файл | `--passphrase-file /path/to/key` | CI / скрипты |

Для HTTP API пароль передаётся в заголовке `X-Mnemos-Passphrase` —
никогда в теле запроса — поэтому он не попадает в логи как параметр
запроса.

## Инкрементальные бэкапы

Используйте `--since`, чтобы экспортировать только памяти, созданные или
обновлённые после указанной временной метки. Это основа для периодических
бэкапов:

```bash
# Ежедневный инкремент — только памяти, затронутые со вчера
mnemos export --format json --since "$(date -u -d 'yesterday' +%Y-%m-%d)" \
  --output daily-$(date -u +%Y-%m-%d).json
```

Комбинируйте с `--compress gzip` и `--encrypt` для компактного,
защищённого ежедневного снимка.

## Режимы импорта

| Режим | Флаг | Поведение | Деструктивный? |
|-------|------|-----------|----------------|
| **merge** | `--mode merge` (по умолчанию) | Вставляет памяти с отсутствующим ID; пропускает существующие (или обновляет с `--overwrite`). Проекты сливаются. Векторы регенерируются для опубликованных памятей. | Нет — идемпотентный |
| **restore** | `--mode restore --confirm` | Стирает все памяти, векторы и проекты, затем импортирует. Для SQLite сырые файлы БД заменяются (после опционального бэкапа). | **Да — требует `--confirm`** |
| **dry-run** | `--dry-run` | Валидирует файл экспорта без записи. Работает с обоими режимами. | Нет |

### merge — идемпотентный

```bash
mnemos import backup.json --mode merge
```

Памяти, чей ID уже существует в целевом хранилище, **пропускаются** по
умолчанию. Передайте `--overwrite`, чтобы обновить их импортированной
версией. Проекты сливаются: проект создаётся, если отсутствует, или его
пути обновляются, если изменились.

### restore — деструктивный

```bash
mnemos import backup.json --mode restore --confirm
```

Режим restore **удаляет все существующие памяти, векторы и проекты**
перед импортом. Это нельзя отменить. Команда отказывается выполняться
без `--confirm` и выводит предупреждение о том, что будет потеряно.

Для SQLite-восстановления можно сначала создать резервную копию текущей
БД:

```bash
mnemos import snapshot.tar.gz --mode restore --confirm --backup-dir ./pre-restore
```

### dry-run — сначала валидация

```bash
mnemos import backup.json --mode merge --dry-run
mnemos import backup.json --mode restore --dry-run
```

Валидирует файл экспорта (формат, схема, читаемость) и сообщает, сколько
памятей будет импортировано / пропущено / обновлено — без записи.
Всегда запускайте это перед `restore`.

## Версионирование формата

Каждый JSON-экспорт несёт две версионные метки в метаданных:

| Поле | Значение |
|------|----------|
| `format_version` | Версия схемы экспорта (currently `1.0`). Увеличивается при ломающем изменении JSON-структуры. |
| `mnemos_version` | Версия Mnemos, создавшей экспорт. |

При импорте Mnemos проверяет `format_version` и предупреждает, если
схема не распознана. Это обеспечивает прямую совместимость — будущий
Mnemos сможет импортировать экспорт `1.0` даже после эволюции схемы.

## Справочник CLI

### `mnemos export`

```bash
mnemos export [OPTIONS]
```

| Опция | По умолчанию | Описание |
|-------|--------------|----------|
| `--output`, `-o` | `mnemos-export.json` | Путь к файлу вывода |
| `--format`, `-f` | `json` | `json` или `sqlite` |
| `--compress` | `none` | `none`, `gzip`, `zstd` |
| `--encrypt` | выкл | Шифровать паролем (AES-256-GCM) |
| `--passphrase-file` | (ввод) | Читать пароль из этого файла |
| `--project` | (все) | Фильтр по проекту |
| `--agent` | (все) | Фильтр по агенту |
| `--status` | (все) | Фильтр по статусу памяти |
| `--tags` | (все) | Теги через запятую (логика И) |
| `--since` | (все) | Только памяти после этой ISO-даты |
| `--until` | (все) | Только памяти до этой ISO-даты |
| `--dry-run` | выкл | Валидировать ввод без записи |
| `--config`, `-c` | (авто) | Путь к config.yaml |

### `mnemos import`

```bash
mnemos import SOURCE [OPTIONS]
```

| Опция | По умолчанию | Описание |
|-------|--------------|----------|
| `SOURCE` (аргумент) | обязательно | Файл экспорта для импорта |
| `--mode`, `-m` | `merge` | `merge` или `restore` |
| `--overwrite` | выкл | Обновлять существующие памяти в merge |
| `--confirm` | выкл | Подтвердить деструктивный restore (обязательно для `restore`) |
| `--dry-run` | выкл | Валидировать без записи |
| `--passphrase-file` | (ввод) | Читать пароль расшифровки из файла |
| `--backup-dir` | (нет) | Бэкап текущей БД сюда перед restore |
| `--config`, `-c` | (авто) | Путь к config.yaml |

## HTTP API

### `POST /api/v1/export`

Стримит экспорт как файл для скачивания.

**Тело запроса** (`application/json`):

```json
{
  "format": "json",
  "compress": "gzip",
  "encrypt": false,
  "project": "mnemos",
  "agent": null,
  "status": null,
  "tags": null,
  "since": null,
  "until": null
}
```

| Поле | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `format` | string | `"json"` | `json` или `sqlite` |
| `compress` | string | `"none"` | `none`, `gzip`, `zstd` |
| `encrypt` | boolean | `false` | Шифровать паролем |
| `project` | string\|null | `null` | Фильтр по проекту |
| `agent` | string\|null | `null` | Фильтр по агенту |
| `status` | string\|null | `null` | Фильтр по статусу |
| `tags` | array\|null | `null` | Теги для фильтра (логика И) |
| `since` | string\|null | `null` | Нижняя граница ISO-даты |
| `until` | string\|null | `null` | Верхняя граница ISO-даты |

**Пароль шифрования** — передавайте через заголовок `X-Mnemos-Passphrase`.
Если `encrypt: true` и заголовок отсутствует, эндпоинт вернёт `400` с
`{"detail": "Encryption requested but X-Mnemos-Passphrase header is missing."}`.

**Ответ** — `StreamingResponse` с `Content-Disposition:
attachment; filename="mnemos-export.<suffix>"`. Суффикс зависит от
формата + сжатия + шифрования (`json`, `json.gz`, `tar.gz`, `enc`).

### `POST /api/v1/import`

Загружает файл экспорта как multipart form data и импортирует его.

| Параметр | Расположение | Тип | По умолчанию | Описание |
|----------|--------------|-----|--------------|----------|
| `file` | form (обязательно) | file | — | Файл экспорта |
| `mode` | query | string | `merge` | `merge` или `restore` |
| `overwrite` | query | bool | `false` | Обновлять существующие в merge |
| `confirm` | query | bool | `false` | Обязательно для `restore` |
| `dry_run` | query | bool | `false` | Валидировать без записи |
| `X-Mnemos-Passphrase` | header | string | (нет) | Пароль расшифровки |

**Ответ** (`200 OK`):

```json
{
  "imported": 142,
  "skipped": 3,
  "updated": 0,
  "errors": [],
  "warnings": [],
  "format_version": "1.0",
  "mnemos_version": "2.0.0"
}
```

## Типовые сценарии

### Полный зашифрованный бэкап (CLI)

```bash
mnemos export --format sqlite --compress gzip --encrypt \
  --output backup-$(date -u +%Y%m%d).tar.gz.enc
```

### Восстановление из бэкапа (CLI)

```bash
# 1. Сначала валидация
mnemos import backup-20260620.tar.gz.enc --mode restore --dry-run

# 2. Бэкап текущего состояния, затем восстановление
mnemos import backup-20260620.tar.gz.enc --mode restore --confirm \
  --backup-dir ./pre-restore-$(date -u +%Y%m%d)
```

### Миграция проекта между инстансами

```bash
# Исходный инстанс
mnemos export --format json --project mnemos --output mnemos-project.json

# Целевой инстанс
mnemos import mnemos-project.json --mode merge
```

### Периодический инкрементальный бэкап (cron)

```cron
15 3 * * *  mnemos export --format json --compress gzip --since "$(date -u -d 'yesterday' +%Y-%m-%d)" --output /backups/mnemos-$(date +\%Y\%m\%d).json.gz
```
